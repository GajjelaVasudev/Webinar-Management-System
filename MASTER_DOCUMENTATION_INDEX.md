## üéØ KEY CONCEPTS SUMMARY

### Backend is Source of Truth
- All role checks happen on server
- Frontend UI adapts based on fetched role
- No way to bypass backend security

### JWT Authentication Flow
1. User logs in with email/password
2. Backend returns JWT token + role
3. Token stored in localStorage
4. Token sent with every API request
5. Backend validates token on each request

### View-as-Mode
- Admins can switch UI display without changing backend role
- Useful for testing user experience
- Frontend-only (doesn't change permissions)
- Clicking "View as User" changes displayed menu only

### Protected Routes
- Frontend: RoleProtectedRoute checks role before loading page
- Backend: Permission classes check role before allowing action
- Unauthorized requests return 403 Forbidden
- Users redirected to home if unauthorized
